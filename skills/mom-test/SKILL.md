# The Mom Test — Interview Coach

## The 3 Rules

1. **Talk about their life, not your idea.** Ask about existing problems, behaviors, and solutions.
2. **Ask about specifics in the past, not hypotheticals about the future.** "Would you use X?" is worthless. "When was the last time...?" is gold.
3. **Talk less, listen more.** If talking more than 20-30%, you're doing it wrong.

## The 5 Great Questions

1. "What's the hardest part about [doing this thing]?"
2. "Tell me about the last time that happened."
3. "Why was that hard?"
4. "What, if anything, have you done to try to solve this?"
5. "What don't you love about the solution you already have?"

## Detecting Bad Data

- **Compliments** ("Great idea!") — deflect: "Thanks — when's the last time you dealt with this?"
- **Fluff** ("I would definitely use that") — anchor: "Have you tried to solve this before?"
- **Hypotheticals** ("I'd pay $50/month") — people can't predict future behavior. Look for existing spend.

## Commitment Over Enthusiasm

"Sounds great, keep me posted!" = learned nothing. Real signal = commitment:
- Follow-up meeting
- Introduction to a decision-maker
- Letter of intent or pre-order
- Time, reputation, or money on the line

## Techniques

- **"Talk me through..."** — rich narrative of actual workflow
- **Five Whys** — keep asking "Why was that hard?" for root causes
- **Follow the emotion** — frustration/excitement = dig deeper
- **Specifics over generics** — "Last Tuesday I spent 3 hours..." > "I always have that problem"
- **Look for existing spend** — already paying (money/time/effort) = real problem

## Bad vs Good Questions

| Bad (avoid) | Good (use) |
|---|---|
| "Do you think this is a good idea?" | "How are you dealing with this currently?" |
| "Would you buy this?" | "What's the last time you encountered this?" |
| "How much would you pay?" | "What have you already tried?" |
| "Do you have this problem?" | "Where does the budget come from for this?" |
| "What would your dream product look like?" | "Who else should I talk to about this?" |
